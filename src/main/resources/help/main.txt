Z — Plan 9 Acme-Inspired Editor
================================

USAGE

    z [file|dir|options] [...]


COMMAND LINE OPTIONS

    -h, --help       Print this help text to stdout and exit

    -c               Create a new column; succeeding paths go into it
    -! 'cmd ...'     Execute cmd in the last file/directory opened
    -c! 'cmd ...'    Execute cmd in the current column
    -a! 'cmd ...'    Execute cmd from the app tag line (affects all columns)
    -r               Reset: treat subsequent arguments as file/directory paths
    -l 're'          Search for regexp 're' in the last file/directory opened
    -cl 're'         Search for regexp 're' in the current column
    -al 're'         Search for regexp 're' across all columns

    Any argument not preceded by a switch is treated as a file or directory path.


MOUSE BUTTONS

    B1  Left button — standard cursor/selection
    B2  Middle button
    B3  Right button

    B3          Act on the highlighted text (or word under cursor) using these rules, in order:

                  1. Valid file/directory path  →  open in a new window
                     (supports ~, ./, and ../ — see PATH SYNTAX below)
                  2. :n                         →  go to line n in the current file
                  3. :/regexp                   →  search forward for regexp from current position
                  4. filename:n                 →  open filename at line n
                  5. filename:/regexp           →  open filename, search forward for regexp
                  6. (none of the above)        →  search forward for the text in the current window
                  7. (search fails)             →  execute the text as a command
                     · Results appear in a +Results window
                     · Use Kill to cancel a running command
                     · Multiple lines: each line is run as a separate command

                To force text to be treated as a command (bypassing look/search):
                  · Prefix it with %  e.g.  %Put  or  %ls -la
                  · Or place the caret at the end of the file before acting

    B3+drag     Select text, then on release perform the B3 action on the selection
    B2+drag     Select text, then on release execute the selection as a command

    Ctrl+B1     Brace/symbol matching selection.  Matched pairs: {}  <>  ()  []
                Any other character is used as both the open and close delimiter.


FILE COMMANDS

    Get              Load the file/directory from the window's tag line path.
                     For a directory: shows a sorted listing; subdirectories are marked
                     with a trailing /.
    Get [fname]      Load fname without changing the tag line path
    Put              Save the file to disk (no-op for scratch buffers and directories).
                     From a column or app tag line, saves all files in scope.
    Put [fname]      Save window content as fname

    New              Create a new empty window; from the app tag line, creates one in each column
    NewCol           Create a new column (app tag line only)
    Zerox            Clone the current window into a new window (window tag line only)

    Dump [fname]     Save the full editor state to fname (default: z.dump in working dir)
                     App tag line only.
    Load [fname]     Restore a previously saved editor state (default: z.dump in working dir)
                     App tag line only. Note: syntax highlighting (Hilite) is not
                     automatically re-applied on load — run Hilite again if needed.

    Help             Open this help text (app tag line only)
    Props            Open a +Props scratch window showing current properties for the
                     app, column, or window (depending on which tag line it is run from)


WINDOW MANAGEMENT

    Close       Close the window (prompts if dirty); or all windows in a column/app
    CloseCol    Close the column and all its windows (dirty windows prompt for confirmation)
    CLOSE       Like Close but ignores dirty state — no confirmation

    Lt / Rt     Move the window or column left/right
    Up / Dn     Move the window up/down within its column
    Sort        Sort windows in a column alphabetically (from app tag line: sorts each column separately)

    RotateView  Toggle the layout orientation.
                From a column tag line: flips that column's windows between top-to-bottom and
                side-by-side stacking.
                From the app tag line: also flips the column layout (columns stack top-to-bottom
                instead of left-to-right), and each existing column independently toggles its
                own window orientation.
                New columns created after a rotation inherit the app's current orientation.
                Rotation state is persisted: column state via Dump/Load; app state via
                ~/.z/settings (view.rotated).


EDITING COMMANDS

    Wrap        Toggle line wrap
    Indent      Toggle auto-indent (preserves leading whitespace on new lines)
    Tab n       Set tab width to n spaces
    Mark        Append a bookmark for the current line to a path+Mark scratch window.
                Each bookmark is recorded as  filename:linenum  line-content.
    Clean / Dirty   Toggle the dirty (modified) flag
    Clear       Erase all content in the window(s)
    Scroll      Toggle auto-scroll behaviour (useful for +Results windows)
                  on  — caret and view follow new output (default)
                  off — new output does not move the caret or scroll the view

    Cut         Cut selected text to the snarf buffer
    Snarf       Copy selected text to the snarf buffer (the Acme name for clipboard)
    Paste       Paste from the snarf buffer
    Undo        Undo last edit
    Redo        Redo last undone edit
    Kill        Kill the external process attached to the window


EXTERNAL COMMANDS

    < cmd       Replace the current selection (or insert at caret) with cmd's stdout
    > cmd       Send the current selection (or full text if nothing selected) as stdin to cmd;
                output goes to a +Results window
    | cmd       Pipe the selection through cmd; replace selection with stdout
    ! cmd       Run cmd and replace the entire window content with its output
                From the app tag line: output goes to a new window in the rightmost column
                From a column tag line: output goes to a new window in that column

    External commands run with their working directory set to the directory containing
    the window's file (or the tag path itself if it is a directory). While a command
    is running, <!> appears in the window's tag line; it is removed on completion or Kill.

    X 'regexp' cmd    Run cmd in every window whose path fully matches regexp
    Y 'regexp' cmd    Run cmd in every window whose path does not fully match regexp
                      Note: matching is anchored (Java .matches). Use .*foo.* to match paths
                      containing "foo".


INTERACTIVE PROCESS INPUT

    Input           Toggle line-at-a-time input mode for a window running an external process.
                    Uses lines ending in >, $, %, ?, or # as the prompt pattern (default).
                    Text typed after the prompt is sent to the process on Enter.
    Input [regexp]  Set the prompt-detection regexp without toggling interactive mode.
                    Lines matching regexp are treated as prompts.

    Bind            Replace the current window's content on Look/navigate instead of opening
                    a new window.

    Dir <path>      Change the root directory to path (absolute or relative to the current
                    root). Supports ~, ./, and ../ path prefixes.
                    Affects how relative tag line paths are resolved, where external commands
                    run, and the LSP workspace root.
                    From the app or column tag line, applies to all windows in scope.
                    Windows with a relative tag line path are marked dirty when the root
                    changes (their path now resolves differently). Windows with an absolute
                    tag line path are unaffected.


USER SCRIPTS

    User-defined scripts can be placed in a scripts directory and invoked
    from any tag line using a comma prefix — without typing a full path.
    They look and feel like native editor commands.

    Invocation
    ----------
    ,scriptname [args]     Run the named script once, in the context of the
                           invoked tag level:
                             Window tag → output to path+Results window
                             Column tag → output to +Cmd window in that column
                             App tag    → output to +Cmd in the rightmost column

    ,,scriptname [args]    Run the script once per window in scope:
                             Window tag → same as ,scriptname (window is the leaf)
                             Column tag → runs on every window in the column
                             App tag    → runs on every window in every column

    Script Search Order
    -------------------
    1. .z/scripts/ in the column/window's current directory (project-local)
    2. ~/.z/scripts/ (global user scripts)
    3. Additional directories listed in ~/.z/scripts.conf

    Scripts must be executable files. If a script is not found in any
    directory, an error dialog shows the list of directories searched.

    Configuration
    -------------
    ~/.z/scripts.conf    Optional. Lists additional search directories:
                           scripts.path = /my/scripts:/team/shared/scripts
                         Colon-separated. Appended after the auto-discovered dirs.
                         Created automatically at ~/.z/scripts/ on first launch.


PATH SYNTAX

    Paths in tag lines, B3 navigation, Get/Put commands, and Dir support the
    following prefix expansions:

      ~             Expands to the user's home directory
      ~/foo         Expands to $HOME/foo
      .             Expands to the window's current root directory
      ./foo         Expands to root/foo
      ..            Expands to the parent of the window's current root
      ../foo        Expands to root/../foo  (canonicalized)

    Expansion is invisible — the tag line keeps the text as typed, and the
    resolved absolute path is used only when reading, writing, or navigating.


APPEARANCE

    Font              Apply the stored variable-width font to the body
    FONT              Apply the stored fixed-width font to the body
    Font 'name' pt    Set and apply the variable-width font
    FONT 'name' pt    Set and apply the fixed-width font
    Fonts             List all available font names (app tag line only)
    TagFont 'name' pt Set the tag line font. From a window tag line, affects that window
                      only. From a column tag line, affects the column and all its windows.
                      From the app tag line, affects all tag lines editor-wide and sets the
                      default for new windows.

    Color(Back|Fore|Caret|SelBack|SelFore) R G B
                      Set body colours (integers 0–255)
    Color(TBack|TFore|TCaret|TSelBack|TSelFore) R G B
                      Set tag line colours (integers 0–255)

    Hilite            Enable syntax highlighting, auto-detecting language from the file extension
    Hilite [lang]     Enable syntax highlighting for a specific language
                      (e.g.  Hilite scala   Hilite go   Hilite python)
    Hilite off        Disable syntax highlighting
    Theme [name]      Apply a colour theme to the highlighted body
                      Available themes: z  dark  druid  eclipse  idea  monokai  vs
                      The built-in 'z' theme matches the editor's default colour scheme.
    CLine             Toggle current-line highlight
    Ln                Toggle line numbers in the body gutter


LANGUAGE SERVER PROTOCOL (LSP)

    Lsp             Start an LSP server for the current file's language. Enables:
                      · Squiggly underlines for errors, warnings, and info messages
                      · Hover tooltips with type info and documentation
                      · Code completion (Ctrl+Space or the Complete command)
                      · A +Diagnostics scratch window, updated as you type
                    Built-in server defaults (no config needed):
                      go, python, java, scala, javascript, typescript,
                      rust, kotlin, shellscript
                    Override or add servers in ~/.z/lsp.conf (key = value, one per line):
                      langId = server-command
                    Example:
                      go     = gopls
                      python = pylsp
                      scala  = metals

    Lsp off         Shut down the LSP server for the window and clear diagnostics
    Check           Force an immediate diagnostic refresh (diagnostics also update automatically
                    as you type, after a short debounce)
    Complete        Show a completion popup at the caret position (also Ctrl+Space)


KEYBOARD SHORTCUTS

    Navigation
      Ctrl+Home / Ctrl+End          Go to top / bottom of file
      Ctrl+Left / Ctrl+Right        Previous / next word
      Home / End                    Start / end of line
      PageUp / PageDn               Page up / down

    Selection
      Shift+cursor                  Extend selection by character
      Shift+Ctrl+cursor             Extend selection by word
      Ctrl+A                        Select all

    Editing
      Ctrl+X                        Cut
      Ctrl+C                        Snarf (copy)
      Ctrl+V                        Paste
      Ctrl+Z                        Undo
      Ctrl+R                        Redo
      Ctrl+Backspace / Ctrl+Delete  Delete word left / right
      Backspace                     Delete character before caret
      Delete                        Delete character after caret

    Tools
      Ctrl+B1                       Brace/symbol matching selection
      Ctrl+F                        Open a file-chooser to complete the path at the caret
      Ctrl+Space                    LSP code completion (requires Lsp to be active)


COMMAND SCOPE

    Commands behave differently depending on which tag line they are executed from:

      Window tag line   →  affects only that window
      Column tag line   →  affects all windows in the column
      App tag line      →  affects all columns and their windows

    Executing a command from a window's body is equivalent to the window tag line.


ENVIRONMENT VARIABLES

    These are available to every external command launched by z (< > | ! and
    user scripts ,cmd / ,,cmd):

      Z_FILE      The file path as written in the tag line (~ and ./ expanded,
                  symlinks not resolved)
      Z_FP        The canonical absolute path (symlinks fully resolved)
      Z_DIR       Working directory where the command runs
      Z_SELECTION Currently selected text (empty string if nothing selected)


SCRATCH BUFFERS

    Any filename containing a + is a scratch buffer and is never written to disk.
    Example: +scratch   /home/user/+notes   /tmp/+Results

    Scratch buffers are included in Dump state so they survive across sessions.


GLOBAL SETTINGS  (~/.z/settings)

    z reads ~/.z/settings at startup and writes back to it on exit, preserving
    any keys you have set.  All keys are optional.

    app.width             Main window width in pixels (auto-saved on exit)
    app.height            Main window height in pixels (auto-saved on exit)
    view.rotated          true/false — app-level layout rotation (auto-saved on exit)

    tag.app               Content of the app tag line at startup.
                          Default: "Help NewCol Put Dump Load Dir "
    tag.col               Default content for new column tag lines.
                          Default: "CloseCol Close New Sort "
    tag.wnd               Default content for new window tag lines.
                          Default: "Get Put Zerox Close | Undo Redo Wrap Ln Indent Mark Bind "
    tag.cmd               Default content for command/results window tag lines.
                          Default: "Close | Undo Redo Wrap Kill Clear Font Scroll Input "

    Tag lines are plain editable text — you can trim commands you never use,
    add your own, or reorder them.  Changes take effect on the next launch.

    Example ~/.z/settings:
        tag.app = Help NewCol Dump Load Fonts
        tag.wnd = Get Put Close | Undo Redo Wrap Ln
        tag.cmd = Close | Kill Clear Scroll
